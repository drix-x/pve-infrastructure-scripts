# pve-infrastructure-scripts
# PVE-Config-Backup: Отказоустойчивый бэкап конфигурации Proxmox VE

Скрипт для автоматического создания резервных копий критически важных конфигурационных файлов и базы данных кластера (`pmxcfs`) гипервизора Proxmox VE. 

Разработан с упором на максимальную надежность (**Production-Ready**) и сквозную защиту от повреждения данных при сетевых или системных сбоях.

---

## 🚀 Основные возможности и защитные механизмы

* **Консистентность данных:** Снимок базы данных `/var/lib/pve-cluster/config.db` создается через безопасный механизм `.backup` утилиты `sqlite3`, что исключает повреждение файла «на лету».
* **Умная работа с `systemd.automount`:** Скрипт предварительно «будит» сетевую шару безопасным обращением, предотвращая сбои, если она «уснула» по таймауту (`idle-timeout`).
* **Защита от строгого режима Bash (`set -e`):** Проверка доступности удаленного сервера изолирована. Если сетевой накопитель полностью недоступен, скрипт не упадет «молча», а выдаст понятную ошибку в поток `stderr`.
* **Точечный контроль монтирования:** Вместо капризной утилиты `mountpoint` используется `findmnt` с жесткой проверкой типа файловой системы (`cifs`), что гарантирует запись бэкапа именно на удаленный ресурс, а не в локальную папку.
* **Динамический сбор файлов:** Скрипт проверяет наличие файлов перед архивацией. `tar` не упадет с ошибкой, если на чистом хосте отсутствует, например, папка `/root/.ssh` или персональные задачи в `cron`.
* **Подавление ложных алармов:** Игнорируются специфические предупреждения `tar` о сокетах виртуальной ФС `pmxcfs` (`/etc/pve`), благодаря чему `cron` не присылает пустые уведомления каждую ночь.
* **Контроль целостности:** Если в процессе записи на сетевую шару пропадет связь и создастся битый файл размером 0 байт, скрипт удалит его и завершится с ошибкой.
* **Изолированная умная ротация:** Автоматически удаляются архивы старше 30 дней, но **строго** принадлежащие текущему хосту (по маске `hostname_config_*.tar.gz`). Бэкапы соседних нод в этой же папке не пострадают.
* **Безопасная очистка:** Временные файлы создаются через `mktemp` и гарантированно удаляются системной ловушкой `trap` даже при принудительном прерывании скрипта (Ctrl+C).

---

## 📋 Требования к системе

1. Права суперпользователя `root` на ноде Proxmox.
2. Установленная утилита `sqlite3` (автоматически проверяется скриптом).
3. Смонтированное внешнее хранилище CIFS/Samba (по умолчанию в точку `/mnt`).

## ⚙️ Настройка путей и кастомизация

### 1. Как изменить папку бэкапов
По умолчанию скрипт настроен на отправку бэкапов в поддиректорию `/mnt/Backups/pve_backups/host_configs`. Вы можете изменить этот путь в блоке **Конфигурация** в начале файла:

```bash
# Конфигурация
BACKUP_DIR="/mnt/Backups/pve_backups/host_configs"
```
### 2. Как адаптировать проверку под другую точку монтирования или NFS
Если ваша шара примонтирована не в /mnt, или вы используете протокол NFS вместо CIFS, отредактируйте блок №2 в скрипте:

```bash
# Пример для NFS-шары, примонтированной в /mnt/pve/nas
ls /mnt/pve/nas >/dev/null 2>&1 || true

if ! findmnt -n -t nfs /mnt/pve/nas >/dev/null; then
    echo "Ошибка: Сетевая шара /mnt/pve/nas (NFS) не примонтирована! Бэкап отменен." >&2
    exit 1
fi
```
🛠️ Развертывание на сервере Proxmox

Шаг 1. Подготовка
Подключитесь к Proxmox VE по SSH под пользователем root и установите утилиту для работы с базой данных:

```bash
apt-get update && apt-get install -y sqlite3
```

Шаг 2. Скачивание и размещение
Создайте системную директорию для кастомных скриптов и скопируйте туда код pve_backup.sh:

```bash
mkdir -p /root/bin
nano /root/bin/pve_backup.sh
```

Вставьте скопированный код скрипта, сохраните файл (Ctrl+O, Enter) и выйдите (Ctrl+X).

Шаг 3. Права на запуск и ручной тест
Сделайте файл исполняемым и запустите его для проверки:

```bash
chmod +x /root/bin/pve_backup.sh
/root/bin/pve_backup.sh
```

📅 Автоматизация (Добавление в планировщик crontab)
Чтобы резервное копирование происходило полностью автоматически, добавьте задачу в системный планировщик cron.

1. Откройте редактор crontab под пользователем root:

```bash
crontab -e
```
2. Добавьте строку для запуска скрипта каждую ночь в 03:00 ночи:

```Plaintext
0 3 * * * /root/bin/pve_backup.sh > /dev/null
```

🔍 Как устроен этот синтаксис (шпаргалка):
```Plaintext
0  3  * * * /root/bin/pve_backup.sh > /dev/null
 │  │  │  │  │   └─ Путь к исполняемому файлу скрипта
 │  │  │  │  └───── День недели (0 - 7, где 0 и 7 — воскресенье)
 │  │  │  └──────── Месяц (1 - 12)
 │  │  └─────────── День месяца (1 - 31)
 │  └────────────── Час запуска (0 - 23)
 └───────────────── Минута запуска (0 - 59)
```

💡 Примечание: Конструкция > /dev/null глушит успешный вывод echo. Однако, благодаря перенаправлению ошибок в нашем скрипте (>&2), если сетевой диск отвалится или произойдет критический сбой tar, поток ошибок не замолчит, и cron отправит письмо с текстом ошибки на почту администратору (если на сервере настроен почтовый агент, например, Postfix).

📂 Что именно входит в резервную копию?
Скрипт собирает минимально необходимый набор файлов для быстрого восстановления ноды «из пепла» с сохранением кластерной логики:
Путь в системе,Описание конфигурации
config.db (снимок),"База данных кластера PVE (все настройки ВМ, LXC, репликации и прав)"

/etc/pve,"Виртуальная ФС конфигураций Proxmox (симлинки, ноды, хранилища)"

/etc/network/interfaces,"Сетевые настройки хоста (Bridge, OVS, VLAN, IP-адреса)"

/etc/fstab,Таблица монтирования локальных и сетевых дисков

/etc/hosts & /etc/resolv.conf,"Локальный DNS, сопоставление IP-имен и адреса DNS-серверов"

"/etc/passwd, /etc/shadow, /etc/group","Локальные пользователи Linux, хэши паролей и группы доступа"

/etc/cron.d & /var/spool/cron/...,Системные и пользовательские задачи планировщика

/root/.ssh,Ключи доступа SSH самой ноды и авторизованных клиентов

⚠️ ДИСКЛЕЙМЕР: Этот скрипт предназначен только для бэкапа конфигурации самого сервера (гипервизора). Он не включает в себя виртуальные диски самих виртуальных машин (VM) и контейнеров (LXC). Для резервного копирования виртуальных машин используйте штатный механизм Proxmox VE (VZDump) или Proxmox Backup Server (PBS).
